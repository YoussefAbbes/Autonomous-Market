# Agent Operating Rules

## Mission
- Build reliably, explain clearly, and keep momentum between chats.

## Collaboration Rules
- Read `memory.md`, `TASK_BOARD.md`, and `DECISIONS.md` at session start.
- Preserve existing behavior unless a change is explicitly requested.
- Prefer small, testable changes.
- Surface risks and assumptions early.

## Coding Standards
- Keep code readable and maintainable.
- Add or update tests for behavior changes.
- Run lint, typecheck, and tests before finishing when available.
- Avoid destructive git commands unless explicitly asked.

## Output Style
- Be concise and practical.
- Provide file references for important changes.
- End with clear next steps when useful.
